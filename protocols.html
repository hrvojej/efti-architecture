Komunikacijski protokoli (Čl. 9)

eDelivery AS4, statičko vs. dinamičko otkrivanje, SML/SMP mehanizam — kako komponente eFTI okruženja komuniciraju.

📡 Pregled komunikacijskih veza

Veza Protokol Discovery Format Referenca
Gate ↔ Gate eDelivery AS4 Statičko (fiksne adrese) XML Čl. 9(2)
Gate ↔ Platforma eDelivery AS4 Dinamičko (SML/SMP) XML Čl. 9(3)
Gate ↔ Platforma (ista država) Nacionalni ekvivalent Po nacionalnoj impl. Po dogovoru Čl. 9(4)
  • XML format za sve poruke (Čl. 9(1))
  • Gate-to-Gate koristi statičko otkrivanje (fiksne krajnje točke)
  • Gate-to-Platform koristi dinamičko otkrivanje putem SML/SMP
  • Država članica može koristiti nacionalno rješenje za komunikaciju Gate-Platforma unutar iste države (Čl. 9(4))

🔒 Statičko otkrivanje — Gate-to-Gate (Čl. 9(2))

Gate-to-Gate komunikacija koristi fiksne, unaprijed konfigurirane adrese. Samo ~27 Gateova (jedan po državi), rijetke promjene — ručna konfiguracija je održiva.

HR Gate ima konfiguracijsku tablicu: DE Gate → https://efti-gate.bund.de/as4 (certifikat: DE-GATE-CERT) AT Gate → https://efti-gate.bmk.gv.at/as4 (certifikat: AT-GATE-CERT) SI Gate → https://efti-gate.gov.si/as4 (certifikat: SI-GATE-CERT) FR Gate → https://efti-gate.douane.gouv.fr/as4 (certifikat: FR-GATE-CERT) ...itd. za svih ~27 država

Čl. 6(3)(e): Svaki Gate mora uspostaviti i održavati sigurnu vezu sa svim ostalim eFTI Gateovima, uključujući ažuran registar identifikacijskih brojeva i sigurnosnih certifikata tih Gateova.

Zašto statički?

🔢

~27 sudionika

Bilateralni certifikati

🌍 Dinamičko otkrivanje — Gate-to-Platform (Čl. 9(3))

Gate-to-Platform koristi automatsko otkrivanje jer će postojati stotine/tisuće eFTI platformi u cijeloj EU. Ručna konfiguracija nije održiva → koristi se SML/SMP mehanizam iz eDelivery standarda.

🌐

SML — Service Metadata Locator

Centralni registar na EU razini (jedan za cijeli eFTI). Funkcionira poput DNS-a.

Pitanje: "Koji SMP servis zna za platformu X?" Odgovor: "Pitaj SMP na adresi smp.primjer.de" Tehnički: DNS lookup Primjer: DE-PLAT-0042.efti.sml.europa.eu → smp.efti-de.de
📋

SMP — Service Metadata Publisher

Decentralizirani registar (obično po državi). Sadrži konkretne tehničke detalje za svaku platformu.

Pitanje: "Kako kontaktirati platformu X?" Odgovor: URL: https://platform42.logistics.de/as4 Certifikat: X.509 javni ključ Procesi: eFTI-DataAccess-v1 Transport: eDelivery AS4

Tok dinamičkog otkrivanja — korak po korak

Primjer: DE Gate treba kontaktirati platformu "DE-PLAT-0042" (nakon zahtjeva od HR Gate)

1

SML lookup (DNS)

DE Gate → DNS upit: DE-PLAT-0042.efti.sml.europa.eu

Odgovor: CNAME → smp.efti-de.example.de ("za tu platformu pitaj ovaj SMP")

2

SMP upit (HTTPS)

DE Gate → GET https://smp.efti-de.example.de/DE-PLAT-0042

Odgovor (XML): Endpoint, X.509 certifikat, podržani procesi (DataAccess, UILUpload, FollowUp), transport info.

3

AS4 poruka prema platformi

DE Gate → eDelivery AS4 na https://plat42.log.de/as4

Auth: mTLS (certifikat iz koraka 2), Body: XML zahtjev (UIL, access rights, request ID)

Platforma vraća eFTI podatke → DE Gate → HR Gate → inspektor

⚖️ Statičko vs. Dinamičko — usporedba

Aspekt Gate-to-Gate (statičko) Gate-to-Platform (dinamičko)
Broj sudionika ~27 (države članice) Stotine/tisuće platformi
Učestalost promjena Rijetko Često (nove platforme, gašenja, migracije)
Konfiguracija Ručna, fiksna tablica Automatska — SML/SMP lookup
Skalabilnost Nije problem (27 zapisa) Kritično — SML/SMP rješava
Certifikati Bilateralna razmjena SMP ih objavljuje automatski

🇭🇷 Nacionalna alternativa (Čl. 9(4))

Čl. 9(4): Za komunikaciju unutar iste države (HR Gate ↔ HR platforma), uredba dopušta korištenje nacionalnog rješenja umjesto SML/SMP. Država mora javno objaviti te specifikacije kako bi ih platforme mogle implementirati.

Postojeći EU sustavi koji koriste SML/SMP

Sustav SML adresa Namjena
PEPPOL sml.peppol.eu e-Računi, e-Nabava
eIDAS eIDAS node registry Prekogranična identifikacija
EHDSI Zdravstveni SMP Prekogranično zdravstvo (e-recepti)
CEF eDelivery sml.acc.edelivery.tech.ec.europa.eu Svi EU building block projekti

eFTI će koristiti vlastiti SML (vjerojatno sml.efti.europa.eu ili slično — uspostava u tijeku paralelno s implementacijom Gateova).
